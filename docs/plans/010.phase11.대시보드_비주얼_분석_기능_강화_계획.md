## 4. 차트 구성 상세

### 4.1 통계 카드 (개선)

**새로 추가할 카드:**
- **본사 총 이익**: 모든 지부 판매에서 본사가 얻은 마진 합계
- **평균 마진율**: 본사 이익 / 총 판매액 * 100

**계산 로직:**
```typescript
// 본사 이익 = (판매가 - 지부 원가) * HQ 마진율
// 예: 판매가 10,000원, 지부 원가 6,000원, HQ 마진 10%
// → 본사 이익 = (10,000 - 6,000) * 0.10 = 400원

interface SaleWithProfit {
  sale_date: string
  location_id: string
  location_name: string
  product_id: string
  product_name: string
  qty: number
  unit_price: number // 판매 단가
  total_amount: number // 판매 총액
  branch_cost: number // 지부 원가 (pricing_configs에서)
  hq_margin_pct: number // 본사 마진율
  branch_margin_pct: number // 지부 마진율
  hq_profit: number // 계산된 본사 이익
  branch_profit: number // 계산된 지부 이익
  currency: string
}
```

**데이터 가져오기 (Supabase):**
```sql
SELECT
  s.sale_date,
  s.location_id,
  l.name as location_name,
  s.product_id,
  p.name as product_name,
  s.qty,
  s.unit_price,
  s.total_amount,
  s.currency,
  pc.local_cost as branch_cost,
  pc.hq_margin_pct,
  pc.branch_margin_pct,
  -- 본사 이익 계산: (판매가 - 지부원가) * HQ마진 / (HQ마진 + 지부마진)
  (s.total_amount - (s.qty * pc.local_cost)) * pc.hq_margin_pct / (pc.hq_margin_pct + pc.branch_margin_pct) as hq_profit,
  -- 지부 이익
  (s.total_amount - (s.qty * pc.local_cost)) * pc.branch_margin_pct / (pc.hq_margin_pct + pc.branch_margin_pct) as branch_profit
FROM sales s
JOIN locations l ON s.location_id = l.id
JOIN products p ON s.product_id = p.id
JOIN pricing_configs pc ON s.product_id = pc.product_id
  AND s.location_id = pc.to_location_id
WHERE s.sale_date >= :start_date
  AND s.sale_date <= :end_date
ORDER BY s.sale_date DESC
```

## 7. 수익 계산 로직 상세

### 7.1 가격 구조 복습

```
판매가 (Final Price) = 지부 원가 / (1 - 본사마진% - 지부마진%)

예:
- 지부 원가: 6,000원
- 본사 마진: 10%
- 지부 마진: 30%
- 판매가 = 6,000 / (1 - 0.40) = 10,000원
```

### 7.2 이익 분배 계산

```typescript
// 총 마진 = 판매가 - 지부 원가
const totalMargin = finalPrice - branchCost // 10,000 - 6,000 = 4,000원

// 마진율 합계
const totalMarginPct = hqMarginPct + branchMarginPct // 0.10 + 0.30 = 0.40

// 본사 이익 = 총 마진 * (본사 마진율 / 총 마진율)
const hqProfit = totalMargin * (hqMarginPct / totalMarginPct)
// = 4,000 * (0.10 / 0.40) = 1,000원

// 지부 이익 = 총 마진 * (지부 마진율 / 총 마진율)
const branchProfit = totalMargin * (branchMarginPct / totalMarginPct)
// = 4,000 * (0.30 / 0.40) = 3,000원
```

**함수 구현:**
```typescript
export function calculateProfits(
  qty: number,
  unitPrice: number,
  branchCost: number,
  hqMarginPct: number,
  branchMarginPct: number
) {
  const totalAmount = qty * unitPrice
  const totalCost = qty * branchCost
  const totalMargin = totalAmount - totalCost
  const totalMarginPct = hqMarginPct + branchMarginPct

  const hqProfit = totalMargin * (hqMarginPct / totalMarginPct)
  const branchProfit = totalMargin * (branchMarginPct / totalMarginPct)

  return {
    totalAmount,
    totalMargin,
    hqProfit,
    branchProfit,
  }
}
```

## 8. 통화 환산 로직

### 8.1 모든 금액을 KRW로 통일

**이유:** 차트에서 비교하려면 동일 통화 필요

```typescript
export function convertToKRW(
  amount: number,
  currency: Currency,
  exchangeRates: Map<string, number>
): number {
  if (currency === 'KRW') return amount

  const rate = exchangeRates.get(`${currency}_to_KRW`)
  if (!rate) {
    console.warn(`Exchange rate not found for ${currency} to KRW`)
    return 0
  }

  return amount * rate
}
```

**환율 데이터 가져오기:**
```typescript
// 최신 환율 조회
const { data: rates } = await supabase
  .from('exchange_rates')
  .select('*')
  .order('effective_date', { ascending: false })
  .limit(10)

// Map으로 변환
const rateMap = new Map()
rates?.forEach(r => {
  rateMap.set(`${r.from_currency}_to_${r.to_currency}`, r.rate)
})
```
